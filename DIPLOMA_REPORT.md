# Описание программной части дипломной работы GreenFind

## Титульная информация

**Тема:** Разработка поисково-аналитической веб-системы GreenFind  
**Тип работы:** Дипломная работа, программная часть  
**Назначение:** Поиск информации в вебе, просмотр истории запросов, получение аналитики и администрирование системы  
**Технологический стек:** Angular, Node.js, Express, MySQL, HTML, SCSS, TypeScript

---

## Введение

Дипломная работа GreenFind представляет собой веб-приложение для поиска и анализа информации в сети. Система ориентирована на удобный ввод поискового запроса, отображение результатов, сохранение истории обращений и предоставление административной аналитики. Программная часть проекта включает клиентский интерфейс, серверное API, работу с базой данных и интеграцию с внешним поисковым сервисом.

В ходе разработки была поставлена задача создать приложение, которое сочетает в себе привычный поиск и дополнительные возможности для анализа поведения пользователей. Это позволило объединить функциональность поисковой системы и внутренней панели управления, где можно отслеживать динамику запросов, популярные темы и активность пользователей.

В качестве платформы была выбрана архитектура "frontend + backend + database". Клиентская часть отвечает за отображение интерфейса и взаимодействие с пользователем, серверная часть обрабатывает запросы и связывает модули системы, а база данных хранит учетные записи, историю поиска, служебную информацию и аналитические данные. Дополнительно используется внешний источник поисковой выдачи.

## 1. Назначение и общая идея системы

Основная цель GreenFind заключается в том, чтобы предоставить пользователю единый инструмент для выполнения поисковых запросов и последующего анализа полученных данных. В отличие от обычной поисковой страницы, система дополняется личным кабинетом, историей обращений и административной панелью.

Функционально приложение решает несколько задач:

1. выполнение веб-поиска по введенному запросу;
2. отображение списка найденных результатов;
3. сохранение истории поисковых запросов;
4. просмотр и редактирование профиля пользователя;
5. доступ администратора к аналитике и статистике;
6. взаимодействие с сервером через защищенные API-запросы.

Для дипломного проекта важно показать, что система не просто повторяет стандартный поиск, а содержит логику обработки запросов, хранения данных и аналитики. Поэтому в отчете отдельно описываются клиентские экраны, серверные модули и структура обмена данными между ними.

## 2. Функциональные возможности приложения

Программная часть GreenFind включает несколько групп функций.

### 2.1 Пользовательские функции

Пользователь может зарегистрироваться в системе, войти в свой аккаунт, выполнить поиск, открыть найденные ссылки и просмотреть историю запросов. Также предусмотрена возможность изменения личных данных и настроек профиля.

### 2.2 Административные функции

Администратор получает доступ к сводной аналитике, статистике запросов, спискам пользователей и дополнительным отчетам. Такая часть особенно важна для диплома, поскольку демонстрирует более сложную логику, чем обычный поисковый интерфейс.

### 2.3 Сервисные функции

Система работает с базой данных, кэшем и внешним поисковым сервисом. При повторном обращении к популярным запросам возможно ускорение выдачи за счет кэширования, а статистика обращений используется для построения графиков и отчетов.

## 5. Серверная часть

### 5.1 Назначение backend

Серверная часть построена на Node.js и Express. Она обрабатывает запросы от клиента, выполняет бизнес-логику, обращается к базе данных и формирует ответы для frontend.

Основные функции backend:

1. авторизация и проверка доступа;
2. обработка поисковых запросов;
3. работа с историей;
4. получение и сохранение аналитических данных;
5. взаимодействие с MySQL;
6. обращение к внешнему сервису поиска;
7. выдача данных в формате JSON.

## 9. Используемые технологии

### 9.1 Frontend

Frontend реализован на Angular с использованием TypeScript, HTML и SCSS. Такой набор технологий подходит для построения модульного интерфейса и удобной работы с компонентами.

### 9.2 Backend

Серверная часть использует Node.js и Express. Это позволяет быстро строить REST API, разделять маршруты по функциональности и поддерживать гибкую архитектуру.

### 9.3 Database

Для хранения данных применена MySQL. Реляционная модель удобна для структуры с пользователями, историей и аналитическими данными.

### 9.4 Внешняя интеграция

Для получения поисковой выдачи используется внешний сервис. Это дает возможность сосредоточиться на логике приложения, аналитике и пользовательском интерфейсе, не строя поисковый движок с нуля.

## 10. Заключение

Проект демонстрирует полный цикл взаимодействия: от ввода запроса до сохранения истории и анализа результатов.

Программная часть приложения показывает применение современных веб-технологий, принципов модульной архитектуры и разделения ответственности между компонентами. Реализованная структура позволяет развивать систему дальше, добавляя новые виды поиска, расширенную аналитику, улучшенные фильтры и дополнительные инструменты администрирования.
